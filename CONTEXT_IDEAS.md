## B. 공유 기능

공유는 단계적으로 가는 게 맞음.

**현재 가능한 가장 빠른 방법 → Web Share API**
- 병원 카드에 "공유" 버튼 추가
- 클릭 시 OS 네이티브 공유 시트 열림 (카카오톡, 문자, 메모 등)
- 공유 내용: 병원명 + 전화번호 + 주소 + 리뷰 수
- 별도 URL/서버 불필요, 구현 쉬움

**이후 단계 → 병원별 고정 URL + OG 태그**
- `/hospital/강남구-연세동물병원` 같은 URL
- 카카오톡에 링크 붙여넣으면 미리보기 카드로 나옴
- Vercel Edge Function으로 동적 OG 태그 생성 필요 (복잡)

## J. 포지셔닝 / 전략 메모

> context_idea.md에서 흡수 (2026-04 기준)

### 핵심 차별점
1. **영수증 인증 리뷰** — 네이버/카카오에 없음. 인증 뱃지 + 상위 노출
2. **반려동물이 주인공인 UI** — "보리가 다녀온 병원" 프레임
3. **진료비 투명성** — 실제 영수증 기반. 맘카페에서 항상 물어보는 것
4. **다항목 별점** — 진료비 수준/친절도/시설/대기시간

> ⚠️ 영수증 인증이 허술하면 차별점이 오히려 신뢰도 폭탄. 인증 신뢰성이 콘텐츠 수집보다 먼저.

### 수익 모델 (장기)
- 우수협력병원/이벤트 광고 배너 → 이미 banners 테이블 있음
- 병원 상단 노출 유료화 (트래픽 확보 후)
- Google AdSense 연동 (Google Analytics 붙이면서 같이 진행)

### 콘텐츠 시딩 전략
- 보리/타미 실제 후기 10개 이상 직접 작성
- 베타 유저 2~3명 후기 요청
- 영수증 없는 일반 후기도 허용 → 진입 장벽 낮추기 ✅ (이미 구현됨)
