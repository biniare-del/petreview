## B. 공유 기능

공유는 단계적으로 가는 게 맞음.

**현재 가능한 가장 빠른 방법 → Web Share API**
- 병원 카드에 "공유" 버튼 추가
- 클릭 시 OS 네이티브 공유 시트 열림 (카카오톡, 문자, 메모 등)
- 공유 내용: 병원명 + 전화번호 + 주소 + 리뷰 수
- 별도 URL/서버 불필요, 구현 쉬움

**이후 단계 → 병원별 고정 URL + OG 태그**
- `/hospital/강남구-연세동물병원` 같은 URL
- 카카오톡에 링크 붙여넣으면 미리보기 카드로 나옴
- Vercel Edge Function으로 동적 OG 태그 생성 필요 (복잡)

| # | 아이디어 | 우선순위 | 난이도 | 메모 |
|---|---------|---------|--------|------|
| B1 | Web Share API 공유 버튼 (병원 카드/상세) | 🔴 | ⭐ | 병원명+전화+주소+지도링크 텍스트 공유. 지금 당장 가능 |
| B2 | 병원별 고정 URL 페이지 | 🟡 | ⭐⭐⭐ | SEO + 카카오 OG 미리보기의 전제조건. 규모 커지면 필요 |
| B3 | 카카오톡 채널 공유 (OG 태그) | 🟢 | ⭐⭐⭐ | B2 이후에 가능 |

## C. 마이펫 히스토리

병원 갈 때마다 체중 재는 거 맞음 → 체중 변화 추적이 자연스러운 기능.
현재 리뷰에 pet_name이 연결돼 있어서 방문 이력은 이미 데이터가 쌓이고 있음.

| # | 아이디어 | 우선순위 | 난이도 | 메모 |
|---|---------|---------|--------|------|
| C1 | 마이펫 진료 이력 뷰 | 🔴 | ⭐⭐ | 기존 리뷰 데이터에서 pet_name 기준으로 필터. 별도 테이블 불필요 |
| C2 | 체중 기록 (날짜별) | 🔴 | ⭐⭐ | pet_weight_logs 테이블 추가. 그래프로 시각화하면 임팩트 큼 |
| C3 | 진료/처방 기록 직접 입력 | 🟡 | ⭐⭐ | 리뷰와 별개로 간단한 메모 형태 |
| C4 | 심장사상충 투약 알림 | 🟡 | ⭐⭐⭐ | 투약일 등록 → 간격 설정 → 알림. 일단 "기록"만 먼저, 알림은 PWA 이후 |
| C5 | 예방접종 일정 관리 | 🟡 | ⭐⭐⭐ | C4와 같은 구조 |

> **내 의견**: C1(진료 이력) + C2(체중 기록)를 먼저 구현하면 마이펫이 진짜 건강수첩이 됨. 알림(C4, C5)은 PWA 없이도 "기록"은 지금 가능하고 알림은 나중에 붙이면 됨.

## 권장 실행 순서

### 즉시 (쉽고 임팩트 있음)
1. **A1** — 영수증 없이도 가능 UX 명확화

### 단기
5. **C1** — 마이펫 진료 이력 뷰

7. **F2** — 내 커뮤니티 글 마이페이지 탭

### 중장기
9. **B2 + E1** — 병원별 고정 URL + 상세 페이지
10. **G1 + G2 + D2** — PWA + 예약 알림
